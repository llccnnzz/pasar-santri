# Dokumentasi Dashboard Admin

## Deskripsi Umum

Dashboard Admin adalah pusat kontrol utama **Pasar Santri Marketplace** yang memberikan administrator gambaran menyeluruh tentang performa platform, aktivitas pengguna, dan metrics bisnis penting. Dashboard ini menyediakan analisis real-time dan insights untuk pengambilan keputusan strategis.

---

## Fitur Utama Dashboard

### 1. **Status Cards Overview**
- **Total Users**: Jumlah seluruh pengguna terdaftar
- **Total Shops**: Jumlah toko aktif di marketplace
- **Total Products**: Jumlah produk yang terdaftar
- **Total Orders**: Jumlah pesanan keseluruhan

### 2. **Revenue Analytics**
- Revenue overview dengan grafik interaktif
- Total revenue all-time
- Monthly revenue breakdown
- Daily revenue tracking
- Weekly revenue monitoring

### 3. **Monthly Analytics Chart**
- Grafik pertumbuhan 6 bulan terakhir
- Tracking user growth
- Order volume trends
- Revenue progression
- Comparison analysis

### 4. **User Type Distribution**
- Breakdown berdasarkan role pengguna
- Visual chart user types
- Progress bar distribusi
- Percentage analysis per kategori

### 5. **KYC Status Monitor**
- Pending applications counter
- Approved applications tracking
- Rejected applications summary
- Recent KYC activity feed

### 6. **Recent Activity Feed**
- Recent orders dengan detail lengkap
- Recent users registration
- Recent products listing
- Real-time activity monitoring

## Panduan Penggunaan

### 1. Akses Dashboard

1. Login ke panel admin dengan kredensial administrator
2. Setelah login, dashboard akan langsung ditampilkan
3. Pastikan memiliki permission **"admin-dashboard"**
4. Dashboard akan load dengan data terbaru

### 2. Membaca Status Cards

#### Interpretasi Numbers:
- **Green Numbers**: Trend positif/growth
- **Red Numbers**: Trend negatif/decline  
- **Blue Numbers**: Neutral/stable
- **Large Numbers**: Formatted dengan thousand separator

#### Quick Actions dari Cards:
- Click pada card untuk drill-down ke detail
- Hover untuk tooltip informasi
- Right-click untuk context menu (export, etc)

### 3. Menggunakan Revenue Analytics

#### Time Period Selection:
1. Click dropdown "This Month" 
2. Pilih periode yang diinginkan
3. Chart akan auto-refresh dengan data baru
4. Loading indicator akan muncul saat updating

#### Interacting dengan Chart:
- **Hover**: Lihat data point spesifik
- **Click**: Drill-down ke detail periode
- **Zoom**: Pinch/scroll untuk zoom in/out
- **Export**: Right-click → Export as Image/PDF

### 4. Monitoring KYC Status

#### Pending Applications:
1. Lihat counter "Pending Applications"
2. Click **"View All"** untuk detail lengkap
3. Dari dashboard bisa quick approve/reject
4. Notifikasi real-time untuk aplikasi baru

#### Recent Activity Review:
- Scroll activity feed untuk lihat terbaru

- Status color coding untuk quick assessment
- Timestamp relative (e.g., "2 hours ago")

### 5. Recent Orders Monitoring

#### Quick Order Review:

### 6. User Activity Tracking

#### Recent Users Analysis:
- Monitor user registrations

- Track growth patterns
- Spot suspicious activities

#### Product Listings Monitor:
- New product additions
- Price monitoring
- Shop activity levels

## Performance dan Caching

### 1. Data Caching Strategy

#### Cache Levels:
- **Dashboard Data**: Cache 5 menit
- **Statistics**: Cache 15 menit
- **Charts Data**: Cache 10 menit
- **Recent Activity**: Real-time (no cache)

#### Cache Keys:
```
admin_dashboard_stats_{date}
admin_dashboard_revenue_{period}
admin_dashboard_analytics_{range}
admin_dashboard_recent_{type}
```

### 2. Performance Optimization

#### Query Optimization:
- **Eager Loading**: Preload related data

## Troubleshooting

### Problem: "Dashboard loading lambat"
**Diagnosis**:
- Check database performance
- Monitor cache effectiveness
- Verify server resources

**Solusi**:
1. Clear dashboard cache: `php artisan cache:clear`
2. Optimize database queries
3. Increase cache duration untuk data statis
4. Contact sistem administrator jika issue berlanjut

### Problem: "Data tidak update real-time"
**Diagnosis**:
- Check cache configuration
- Verify auto-refresh settings
- Monitor background jobs

**Solusi**:
1. Manual refresh browser (F5)
2. Clear browser cache
3. Check cache expiration settings
4. Restart queue workers jika diperlukan

### Problem: "Charts tidak tampil"
**Diagnosis**:
- Check JavaScript console errors
- Verify chart library loading
- Monitor API endpoints

**Solusi**:
1. Refresh halaman
2. Clear browser cache
3. Check internet connection
4. Verify chart data availability

### Problem: "Permission denied accessing dashboard"
**Diagnosis**:
- Check user role dan permissions
- Verify authentication status
- Monitor access logs

**Solusi**:
1. Verify user memiliki role "admin"
2. Check permission "admin-dashboard"
3. Re-login jika session expired
4. Contact super admin untuk role assignment

### Problem: "Revenue numbers tidak akurat"
**Diagnosis**:
- Check order status filters
- Verify calculation logic
- Monitor data consistency

**Solusi**:
1. Verify dengan database langsung
2. Check order status definitions
3. Recalculate dengan manual query
4. Report ke development team jika ada bug

## Informasi Teknis

### Database Tables Used:
```sql
-- Main entities
users, shops, products, orders, kyc_applications

-- Analytics queries
SELECT COUNT(*) FROM users WHERE role = 'seller'
SELECT SUM(total_amount) FROM orders WHERE status = 'delivered'
SELECT * FROM kyc_applications WHERE status = 'pending'
```

### API Endpoints:
```
GET /admin/dashboard              - Dashboard view
GET /admin/dashboard/analytics    - Analytics data
GET /admin/dashboard/refresh      - Refresh data
POST /admin/dashboard/settings    - Update settings
```

### JavaScript Libraries:
- **ApexCharts**: Chart rendering
- **Chart.js**: Alternative charts
- **DataTables**: Table enhancements
- **Moment.js**: Date formatting

### Cache Configuration:
```php
// Cache configuration
'dashboard_cache_ttl' => 300, // 5 minutes
'analytics_cache_ttl' => 900, // 15 minutes
'stats_cache_ttl' => 600,     // 10 minutes
```
